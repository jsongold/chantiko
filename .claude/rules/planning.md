# Planning Workflow

## CRITICAL: Approval Required Before Implementation

After creating a plan, you MUST stop and wait for explicit user approval.
NEVER begin writing or modifying code until the user confirms.

```
✅ Allowed after planning:  ask questions, refine the plan
❌ Not allowed:             editing files, creating files, running build commands
```

## Required Workflow

1. **Plan** — Use the planner agent or draft a step-by-step plan
2. **STOP** — Present the plan and explicitly ask: "Does this look good? Should I proceed?"
3. **Wait** — Do not take any implementation action until the user says yes (e.g. "go", "looks good", "proceed", "yes")
4. **Implement** — Only after receiving explicit approval

## What Counts as Approval

Explicit approval words: "go", "yes", "proceed", "looks good", "do it", "start", "implement"

When in doubt, ask again rather than assume approval.
